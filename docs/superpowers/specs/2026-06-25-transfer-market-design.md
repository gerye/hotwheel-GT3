# 转会市场(车辆市场)设计文档

> 状态:已与用户在 brainstorming 中确认主干(2026-06-25)。本文件为实现前的权威设计。
> 关联:依赖现有「赛车状态(未签约/现役/退役)」「每类别最多 2 现役」「车队积分」「赛车 MMR + 赛季末快照」「赛季 结束/开启」机制。

## 1. 背景与目标
现状:车队大多按 F1 现实建立,很多收藏的 F1 车模、公路车没有对应车队;这些「自由」车辆无法流动起来。
目标:引入**转会市场**——依据上赛季表现给车队**预算**、给赛车**薪资**,在每个赛季之间让车队在预算内重组阵容,使自由车辆流动,同时保留 F1 那种「强绑定」的真实感。

## 2. 核心概念

### 2.1 合同(新增赛车属性)
- `Car.contract`:`长期` / `短期`(仅当赛车有车队时有意义;未签约车无合同)。
- 由用户按现实设定(参考:F1 约 90% 长期、GT3 约 50%、公路几乎全短期)。在赛车编辑页可改。
- **长期**:转会窗口**不释放**,锁死原队。
- **短期**:窗口**到期解约**,进入自由市场。
- 新签下的自由车手默认 `短期`(用户可改长期)。

### 2.2 车队积分(重新定义,替换旧的 5/4/3、10/8/6)
> 该规则同时用于**荣誉榜**与**预算**。实现时需替换现有 `standings.award_solo/award_team`。
- **单人锦标赛**:每次小组赛**晋级 +1**;最终**冠/亚/季再 +4 / +2 / +1**。
- **车队锦标赛**:同规则但**全部 ×2**。
- 各赛事等级分别叠加;赛季制清零;仅专业赛计分。
- 代表值(单人):冠军 ≈ 6~7、亚军 ≈ 4~5、季军 ≈ 3~4、进决赛圈未上台 ≈ 2、首轮淘汰 0。

### 2.3 预算(车队,整队一个池)
`budget(team) = 800 + 20 × 上赛季车队积分 + 50 × 单人赛冠军次数 + 100 × 车队赛冠军次数`
- 常数:`BUDGET_BASE = 800`、`POINT_VALUE = 20`(每 1 车队积分)、`CHAMP_BUDGET_SOLO = 50`、`CHAMP_BUDGET_TEAM = 100`(车队赛 ×2)。
- 整队一个预算池,覆盖该队**所有现役车**(跨类别)的薪资之和。
- **长期合同是硬性支出**:若仅长期合同薪资就已超预算(余额为负),长期车照常保留,该队只是签不了新人。

### 2.4 薪资(赛车,综合;无负值)
`salary(car) = max(30, 100 + MMR分量 + 100×夺冠次数 + 40×进决赛圈未夺冠次数 + 100×是否名宿)`
- `MMR分量`:赛季末 MMR ≥ 1500 时 `+(MMR−1500)×1`;< 1500 时 `−(1500−MMR)×0.5`(差值的一半作折扣);最终有下限 30,不会为负。
- 常数:`SALARY_BASE = 100`、`SALARY_CHAMP = 100`(每个冠军)、`SALARY_FINALS = 40`(每次进决赛圈未夺冠)、`SALARY_LEGEND = 100`(名宿费,年薪)。
- **名宿**:该车**历史 MMR** 排在**本类别前 10%**(`ceil(10%×该类别车数)`,至少 1 辆)。
- 赛季末 MMR 取已有的 `CarSeasonMMR` 快照;没跑过的车 MMR=1500 → MMR分量 0。
- 数值放 `app/config.py` 可调。

### 2.5 经济平衡(设计目标,已验证)
忽略基础薪资与基础预算(认为互相抵掉),按「一季 2 场、名宿费摊成 名宿费/2 每场」算单场边际收支:

| 名宿车单场 | 支出(薪资边际) | 收益(预算边际) | 净 |
|---|---|---|---|
| 夺冠 | 100 + 50 = 150 | 50 + (6~7)×20 = 170~190 | +20~+40(小赚) |
| 进决赛圈未冠 | 40 + 50 = 90 | (4~5)×20 = 80~100 | −10~+10(大致打平) |
| 首轮淘汰 | 50 | 0 | −50(亏) |

非名宿夺冠:收益 170~190 − 支出 100 = **+70~+90(赚)**。
**效果:名宿夺冠小赚、只进决赛圈大致打平、首轮出局明显亏(−50);非名宿夺冠赚更多。** 买名宿仍需出成绩,早早出局就亏。

以上以**单人赛**为基准;**车队赛各项约 ×2**(车队积分 ×2、夺冠预算 50→100)。车队赛的精确收支不强求,实现后按手感微调。所有常数集中在 `app/config.py`。

## 3. 转会流程(半自动「选秀」)
触发时机:**赛季结束之后、下赛季开启之前**(预算用刚结束赛季的车队积分,薪资用其 MMR 快照)。

步骤:
1. **结算**:为每支队算预算、为每辆车算薪资(存一份赛季快照便于展示与回溯)。
2. **释放**:所有**短期**车解约 → 自由市场池(变 `未签约`);**长期**车留任。
3. **生成推荐阵容(可改)**:反复执行——
   - 取**「预算 − 已占用薪资」余额最高**且仍需补位的车队,**一次把它的阵容签满**(见 §4 规则),用自由池里的车;
   - 重算各队余额,取下一支余额最高的队;直到所有队补满或没有可签的车。
   - 丢了贵价短期车的队余额高 → 自然排在前面优先重建。
4. **你拍板**:在转会页面上可对推荐结果增删改(签某车 / 释放某车 / 换人),系统实时校验约束并显示各队余额。
5. **确认 → 定档**:写入新阵容与合同 → 开启下赛季。

## 4. 阵容与签约规则(校验)
- **每类别 0 或 2 现役**:某类别只要参赛就必须凑满 **2 个现役**(不能买一个空一个);**可整类别不参赛**(钱全砸 GT3、不组公路)。除非自由池里没有满足条件的车,才允许该类别不足 2。
- **品牌锁(铁律)**:厂商车队只能签**本品牌**的车;独立车队不限品牌。**例外:品牌为「无」(空/「无」)的车视为通用车,任何厂商车队都可签约/在推荐中考虑,签约后其品牌保持「无」不变。**
- **预算**:新签约不得使「现役薪资之和」超过预算;**唯一例外**是既有长期合同造成的超支(不因此强制解约长期车)。
- 现役名额上限仍是每类别 2(与现有规则一致)。
- **退役车**:不参与市场、不计薪资、不占现役名额,留在原队(老将);转会只处理现役阵容。

### 4.1 推荐选车规则(可被你覆盖)
某队补某类别到 2 时:在「品牌匹配 + 买得起 + 仍能补齐其余强制空位」的自由车中,优先签**赛季末 MMR 最高**的;预算不足以两人都签高价时,退而求其次选买得起的组合;实在无解则该类别留空或不足 2。

## 5. 首赛季与边界情况
- **首赛季**:无上赛季数据,阵容仍按现状手动建立;**第一次转会发生在第 1 赛季结束后**。届时预算 = 基础(积分为 0),薪资 = 基础 + MMR 快照分量。
- **未签约 / 新收藏车**:都是自由球员,可被选秀签走(默认短期)。
- **没人签的车**:留在 `未签约`。
- **负预算队**:保留长期车、不签新人。
- 转会页面在 `赛季` 区入口,且只有「存在已结束、尚未开启下一季」的间隙时可进入(或显式「开始转会」按钮)。

## 6. 数据模型增改
- `Car` 增 `contract: Optional[ContractType]`(枚举 `长期/短期`,可空);新枚举 `ContractType`。
- 新表 `SeasonMarket`(或按赛季存快照):记录某次转会所属赛季、各队预算、各车薪资,便于展示/回溯。形式:
  - `TeamBudget(season_id, team_id, budget)`、`CarSalary(season_id, car_id, salary)`(随转会结算写入)。
- 新列由现有 `app/db.py::sync_schema` 自动迁移。

## 7. 服务与文件(预期)
- `app/services/market.py`:纯逻辑 + 编排——`compute_budget(team, season)`、`compute_salary(car, season)`、`recommend_rosters(session, season)`(选秀算法)、`sign(session, car, team)` / `release(session, car)`(含约束校验)、`finalize_market(session)`。
- `app/routers/market.py`:转会页面(展示预算/薪资/自由池/推荐阵容,支持增删改与确认)。
- 复用:`standings.team_season_points`、`CarSeasonMMR` 快照、`teams.check_can_assign`(品牌+名额)、`cars.change_status`/`update_car`、季节钩子 `seasons.end_season`/`start_season`。
- 配置:`app/config.py` 增 `BUDGET_BASE/BUDGET_POINT_FACTOR/SALARY_BASE/SALARY_MMR_FACTOR/SALARY_*_BONUS`。

## 8. 测试要点
- 预算/薪资公式(有/无积分、有/无 MMR、冠军加成)。
- 释放:短期解约进池、长期留任。
- 选秀顺序:余额最高的队先签满;丢了贵价车的队优先;品牌锁、预算、每类别 0/2 规则生效;无合适车时允许不足 2。
- 负预算队保留长期车、不签新人。
- finalize 后新阵容/合同正确、可开启下一季。
- 路由:转会页渲染、签约/释放校验、确认。

- 薪资/预算公式仅给默认值,后续按手感调参。
