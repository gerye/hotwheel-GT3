# Hot Wheel GT3 赛事管理系统 — 设计文档

> 状态:已与用户确认(2026-06-22)。本文件是实现前的权威需求与设计说明。

## 1. 项目目标

为家中的风火轮(Hot Wheels)实体跑道搭建一个**网页应用**,登记赛车与车队、组织并记录比赛、维护赛季荣誉(车队积分与赛车 MMR)。跑道共 **4 条赛道**。

## 2. 技术方向

- **运行方式**:本地运行,手机能在同一 WiFi 下打开网址方便录入。界面**响应式**(电脑/手机都好用)。
- **数据存储**:SQLite(单个本地数据库文件 `data/hotwheel.db`,便于备份/搬家);图片存在 `data/images/` 文件夹,数据库存相对路径引用。
- **后端**:Python **FastAPI** + **SQLModel**(SQLAlchemy + Pydantic)+ Uvicorn。
- **前端**:**Jinja2 模板 + HTMX**(无构建步骤,服务端渲染 + 局部动态交互),少量原生 JS/CSS,响应式。
- **测试**:pytest + FastAPI TestClient。
- **启动**:一条命令(`uvicorn app.main:app`)。
- **页面生成**:赛车个人页、车队个人页、赛季页等**不预存**,访问时按模板用数据实时生成。

## 3. 数据模型

### 3.1 赛车 Car
字段:
- 昵称(**唯一**,用户起的、一眼能辨识的名字)
- 图片
- 铸模 casting(车架样子,**可重复**)
- 品牌 brand
- 赛车类别:`F1` / `GT3` / `公路车`(等于赛事等级)
- 叙述
- 所属车队(**可空、可变更**)
- 状态:`未签约` / `现役` / `退役`

系统维护:按类别的赛季 MMR 与历史 MMR、历史战绩/荣誉。

规则:
- 不做铸模/涂装的唯一约束;靠昵称(唯一)+ 图片 + 叙述区分赛车。
- **状态机**(与车队绑定):无车队⟺未签约;有车队⟺现役或退役。
  - 加入车队 → 现役;移出车队 → 未签约;现役 ⇄ 退役经状态控件切换。
  - 禁止「未签约直接→现役」(须先加入车队);「退役→现役」需车队现役名额未满。
  - 退役 → 任意 与 → 未签约 等价于移出车队。
- **专业赛仅限现役**赛车;表演赛任何状态都可。
- 状态控件出现在数据库列表页每行与赛车个人页;加入/移出车队在「编辑」里做。

### 3.2 车队 Team
字段:
- 类型:`厂商车队` / `独立车队`
- 品牌(仅厂商车队有);厂商车队名 = 品牌名 + “车队”
- 名称(独立车队有)
- 各赛事等级的分队别名(F1/GT3/公路,可空)
- 成员赛车

命名(宏观名 vs 具体名):
- **宏观名**(整队,即 `name`):厂商=品牌+「车队」(法拉利车队);独立=用户名称。用于积分榜、数据库列表、泛指整队的场合。
- **具体名**(分等级):厂商=品牌+别名+类别+「车队」(法拉利大王八F1车队);独立=宏观名+别名+类别(车库突击队闪电GT3)。别名为空则省略别名段。用于赛车信息页、具体比赛(如车队锦标赛小组)。别名只在车队页修改。

系统维护:当前赛季车队积分、各赛季历史积分与名次、积分来源明细。

约束(在**设置/变更赛车车队**及**新增车队成员**时即时校验并提示,而非比赛时才报错):
- 厂商车队:下属赛车品牌必须与车队品牌一致。
- 每个赛事类别(F1/GT3/公路)最多 **2 个现役**车手;**退役不占名额**(可有多个退役车手)。

### 3.3 赛季 Season
- 名称/编号、起止、状态(进行中/已结束)。
- 关联:该赛季所有比赛、赛季末各车 MMR 快照、赛季末车队积分榜。
- 操作:结束当前赛季 / 开启下一赛季。

### 3.4 比赛 Race / Tournament
- 所属赛季、赛事等级(F1/GT3/公路)、专业水平(`专业` / `表演`)、赛制类型(`单人锦标赛` / `车队锦标赛`)、参赛车清单、状态。
- 完整轮次与小组结构、每场站位与结果。

### 3.5 单场结果 Heat
- 属于某一轮、某一组;4 个道次分别是哪辆车、名次、是否完赛。

## 4. 数据库页面

### 4.1 统一数据库页
- 顶部「赛车 / 车队」标签切换。
- **全局搜索**:对所有字段生效(昵称、铸模、品牌、车队、叙述等都能搜到)。
- 按类别等筛选。
- 右上角「新增赛车 / 新增车队」入口。
- 列表行:
  - 赛车:缩略图、昵称、铸模/品牌/车队、类别、当前 MMR。
  - 车队:图标、车队名、类型/品牌(或“混合品牌”)、成员车数、当前赛季积分。
- 点击任意行进入对应个人页。

### 4.2 赛车个人页(实时生成)
大图 + 基本信息 + 「编辑」;叙述;赛季 MMR / 类别排名 / 历史 MMR 卡片;历史战绩与荣誉。

### 4.3 车队个人页(实时生成)
车队类型/品牌 + 「编辑」;本赛季积分 / 总榜排名 / 成员数卡片;成员赛车按类别分组并显示占用(如 GT3 2/2);本赛季积分来源逐条;历史各赛季积分榜名次。

## 5. 赛事流程

### 5.1 通用流程
1. 开始新比赛:选赛事等级 → 专业水平(专业/表演)→ 赛制类型(单人锦标赛/车队锦标赛)。
2. 勾选参赛车:系统列出符合该等级、且(专业赛时)有车队的赛车,用户勾选。
3. 随机分组:按规则分组,**可视化展示**每组成员。
4. 站位安排:每组跑 4 场,系统给出每场各道次的车,保证**每辆车把 1-4 道各跑一遍**。
5. 录入结果:每场录入 1-4 名;积分 **5 / 3 / 2 / 1**,未完赛 **0**。非法输入(并列/缺号、名次与未完赛冲突、漏填)红字拦截要求重录。
   - 小组卡片内有**实时积分榜**(行=车,列=第1场…第N场+总分),每录一场刷新并按总分降序;车队赛按车队总分聚合(同队两车相邻,1 车×2),并显示单车小计。
6. 晋级与重组:见各赛制。
7. **回退**:任一录入步骤都可撤销/改写(撤销上一场、修改已录入结果)。

### 5.2 单人锦标赛
- **分组铁律**:每组 **3-4 辆,绝不少于 3**。系统每轮按此自动分组(随机)。能凑成偶数组(晋级数被 4 整除)就凑,凑不成允许奇数组——不强求。
  - 第 1 轮分组算法:把 N 辆车分成若干 3-4 辆的组,优先 4 辆组、必要时用 3 辆组凑齐。
  - 示例:16 → 4/4/4/4;14 → 4/4/3/3;10 → 4/3/3。
- 每组 4 场后按总分,**前 2 名晋级**。
- 2/3 名总分并列 → 弹出 **1V1**,用户直接选择胜者。
- 全部小组晋级完成 → **重新随机分组**,重复。
- 收敛:车数不断减半式收敛,最后只剩一组(3-4 辆)时,**该组 4 场直接决定最终名次**(第 1 名 = 冠军)。
  - 示例:16 → 8 → 4(决赛组);10 → 6 → 4(决赛组)。

### 5.3 车队锦标赛
- 参赛单位是车队;同队的 2 辆(或 1 辆)车**绑定参赛**。
- 每组由 **2 个车队**组成 → 每组 2-4 辆车,跑 4 场,比**车队总分**,总分高的一队晋级。
- 一队 1 车 vs 一队 2 车时:1 车的积分 **×2** 再比较。
- 车队总分并列 → **加赛 4 轮**。

### 5.4 站位算法
每组 4 场,生成一张 4×4(场 × 道)的安排表,保证每辆车在 4 场里把 1/2/3/4 道各跑一次(3 辆车的组同样跑 4 场,遍历分配)。

## 6. 荣誉系统

### 6.1 车队积分(赛季制,赛季结束清零)
- 专业**单人**锦标赛前 3 名 → 该车所属车队 **+5 / +4 / +3**。
- 专业**车队**锦标赛前 3 名 → 车队 **+10 / +8 / +6**。
- 各赛事等级**叠加**。
- 展示:当前赛季总榜;每条积分记录来源(哪辆车 / 哪场比赛),在车队页与赛季页都可见;历史每赛季的积分榜与名次保留。

### 6.2 赛车 MMR
- **按赛事等级分开**:每个等级有独立天梯,只含该等级赛车。
- 每个等级各维护两条:
  - **赛季 MMR**:赛季结束时重置回 1500。
  - **历史 MMR**:永不重置。
- 初始均为 **1500**。
- 仅**专业赛**改变 MMR,在**每个小组 4 场结束后**结算一次(同一辆车在一场锦标赛里可能因晋级多轮而被结算多次)。

#### 算法(Elo 两两对战)
小组 4 场后按总分排名:排你前面的算输、后面的算赢、同分算平。对组内每个对手单独计算:
```
E = 1 / (1 + 10^((对手MMR − 自己MMR) / 400))
S = 赢→1 / 输→0 / 平→0.5
ΔMMR = K × (S − E)            # K = 24
```
本组总变化 = 对组内所有对手 ΔMMR 求和。该公式**零和**(赢家所得 = 输家所失),满足「按分差变动、增减一致」。

赛季 MMR 与历史 MMR **各自独立**用本公式计算(各自天梯、各自分差),因此会随时间分化。

示例(K=24):1500 赢 1500 → 各 ±12;1400 赢 1600(爆冷)→ 赢家约 +18、输家约 −18。

#### 展示
- 各等级的赛季 MMR 榜与历史 MMR 榜。
- 赛车个人页显示当前 MMR 与排名。

## 7. 推荐实现顺序

> 用户选择「全部一次做完」。以下为实现时的建议落地顺序,降低风险,非范围裁剪。

1. 数据层与模型:SQLite schema、Car/Team/Season/Race/Heat、约束校验。
2. 数据库页面:统一列表(含全局搜索/筛选)、新增/编辑、个人页模板。
3. 赛事流程:开始比赛、勾选、分组算法、站位算法、结果录入与回退;先单人锦标赛,后车队锦标赛。
4. 荣誉系统:车队积分结算与展示、MMR(Elo)结算与双天梯、赛季页与赛季结束/开启。
